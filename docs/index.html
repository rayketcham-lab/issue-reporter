Features

Everything built in

A complete feedback workflow in a single JavaScript file.

Multi-Step Wizard

Guided flow: pick a type, add details, review, then submit. Structured data every time.

Element Inspector

Click any element on the page to highlight it. The selector and context are captured automatically.

Console & API Capture

Automatically captures console errors and failed API calls. Attached to the issue for context.

Severity Levels

Critical, high, medium, low. Each report includes a severity tag for triage.

Three Modes

Direct GitHub API, backend endpoint, or GitHub Enterprise. Use the token model that fits.

Security First

Input validation, markdown injection prevention, token scoping, HMAC webhook verification.

Quick Start

Two lines of HTML

Add the widget to any page. Reports go straight to GitHub Issues.

index.html
<!-- Direct GitHub API mode --> <script src="https://cdn.jsdelivr.net/gh/ rayketcham-lab/issue-reporter@main/ issue-reporter.js"></script> <script> IssueReporter.init({ owner: 'your-org', repo: 'your-repo', token: 'ghp_...' }) </script> <!-- Backend mode (recommended) --> <script> IssueReporter.init({ backendUrl: '/api/issues' }) </script>
server.py — Built-in backend
# Start the included backend (stdlib, no Flask) $ python server.py \ --repo your-org/your-repo \ --allowed-origins https://yoursite.com # Rate limiting, CORS, input validation included # Token stays on server — never exposed

How It Works

From click to issue

A guided wizard collects structured feedback and creates a GitHub issue.

1

Select Type

Bug, data issue, UI bug, broken link, feature request, or performance.

2

Add Details

Description, expected behavior, severity. Optionally inspect a DOM element.

3

Review

Preview the issue with auto-captured context: URL, console errors, API failures.

4

Submit

Issue created on GitHub with labels, page context, and browser metadata.

Configuration

Widget options

All options passed to IssueReporter.init().

OptionTypeDescription
ownerstringGitHub org or user (direct mode)
repostringRepository name (direct mode)
tokenstringGitHub PAT with issues:write scope
backendUrlstringBackend endpoint URL (backend mode)
apiUrlstringGitHub API base URL for GHES/GHEC
labelsarrayDefault labels for created issues
positionstringFAB position: bottom-right (default), bottom-left
themestringdark (default) or light
CLI Tools

Command-line interfaces

File issues from the terminal. Both use gh CLI under the hood.

Bash
# Interactive mode $ ./issue-reporter.sh # Quick mode $ ./issue-reporter.sh \ -r org/repo \ -t bug \ -m "Login fails" # Pipe mode $ echo "Crash on submit" | \ ./issue-reporter.sh -r org/repo
Python
# Install $ pip install git+https://github.com/ rayketcham-lab/issue-reporter.git # CLI $ issue-reporter \ -r org/repo \ -t bug \ -m "Login fails" # As a library from issue_reporter import report report("org/repo", "bug", "...")
